## Methodology

The following checks were performed against all files in the working tree and across the full git history (366 commits):

1. **Sensitive file types** -- Searched for `.env`, `.pem`, `.key`, `.p12`, `.pfx`, `.jks`, `.secret`, `credentials`, `id_rsa`, `id_ed25519`, `id_dsa` files.
2. **Private IP addresses** -- Scanned for RFC 1918 addresses (10.x.x.x, 172.16-31.x.x, 192.168.x.x).
3. **API keys and tokens** -- Searched for patterns matching API keys, bearer tokens, and provider-specific key formats (AWS AKIA*, GitHub gh[pousr]_*, Slack xox*, OpenAI sk-*, Anthropic API keys).
4. **Passwords and credentials** -- Searched for password/passwd/pwd assignments and database connection strings with embedded credentials.
5. **Private keys and certificates** -- Searched for PEM-encoded private keys (RSA, DSA, EC, OpenSSH) and certificates.
6. **Cloud provider secrets** -- Searched for AWS, Azure, and GCP secret/key/token patterns.
7. **Authorization headers** -- Searched for hardcoded Bearer tokens and Authorization header values.
8. **Base64-encoded secrets** -- Searched for suspiciously long base64 strings near secret-related keywords.
9. **Git history audit** -- Used `git log -p -S` to search the full commit history for secrets that may have been added and later removed, and `git log --diff-filter=D` to review all deleted files.
10. **Android signing config** -- Checked for signing keys, keystores, or signing passwords in the Android project.

### INFORMATIONAL: Beads credential key in git history (since removed)

- **File:** `.beads/.beads-credential-key` (deleted, no longer in working tree)
- **History:** This 30-byte binary file was committed and removed multiple times between commits `29f94eb` (2026-03-17) and `c5d8b65` (2026-03-20). It was generated by the Beads issue tracker tool and is a local encryption key for the Beads credential store.
- **Risk:** Low. The file is a locally-generated symmetric key for a development tool's local credential store. It is not an API key, cloud secret, or password. The Beads tooling has since been fully removed from the repository. However, if this key was used to encrypt any sensitive material, that material could theoretically be decrypted by someone with access to the git history. Consider running `git filter-branch` or BFG Repo Cleaner to purge it from history if this is a concern.

## Notes on Security Design

- **PSK (Pre-Shared Key):** The console uses a PSK for WebSocket authentication. The PSK is auto-generated at runtime (24-character hex, 96 bits of entropy) and stored in the user's local config directory (`~/.config/dispatch/config.toml`), not in the repository. The `config.default.toml` shipped in the repo has an empty PSK value. Test files use obvious dummy values like `"key"` and `"test-secret-key"`, which is appropriate.
- **TLS certificates:** Generated at runtime as self-signed certs and stored in the user's config directory. No certificates or private keys are present in the repository.
- **`.gitignore` coverage:** The `.gitignore` properly excludes `.dispatch/`, `.dolt/`, `.doltcfg/`, `.claude/`, `logs/`, `*.db`, and Android build artifacts.
